Project Overview

The Customer Data Tools platform streamlines customer support by providing diagnostic tools to quickly identify and resolve data discrepancies between SIS systems (ClassLink/OneRoster) and the Bookmarked platform. This tool eliminates manual database queries and provides actionable remediation suggestions.

Key Features

Multi-Environment Access
Switch between staging and production environments dynamically
Customer Integration Management
Store ClassLink, OneRoster, and FTP configurations per customer
HubSpot Integration
Link diagnostic results directly to support tickets
Secure Credential Storage
Encrypted credentials in AWS Secrets Manager
Read-Only Database Access
Zero risk of modifying production data
Extensible Tool Framework
Easy to add new diagnostic tools as needs arise
Automated Diagnostics
Compare source data vs. production data automatically
Remediation Suggestions
Actionable steps to resolve identified issues

Implementation Phases

Phase 1: Foundation (Oct 14-17)

  • Flask application setup with authentication
  • Database connector implementation (Bookmarked staging/production)
  • Customer integration settings framework
  • Base diagnostic tool class
  • Basic UI with dashboard

Phase 2: Core Tools (Oct 18-22)

  • Student Mismatch Resolver tool
  • Missing Data Finder tool
  • ClassLink API connector
  • OneRoster API/CSV connector
  • Tool UI and results display

Phase 3: Advanced Features (Oct 23-26)

  • Parent Email Conflict tool
  • Campus Transfer Validator tool
  • HubSpot integration (OAuth, ticket import)
  • Automated tool recommendation engine
  • FTP connector for customer data

Phase 4: Deployment & Refinement (Oct 27-31)

  • AWS deployment (Lambda or ECS Fargate)
  • Credential migration to AWS Secrets Manager
  • User acceptance testing with support team
  • Documentation and training materials
  • Performance optimization and monitoring

Success Criteria

Technical Metrics

  • All tools return results in <30 seconds
  • Database connection success rate >99%
  • Zero credential leaks or security incidents
  • Support for all active customer integration types
  • Tool accuracy >95% (validated against known issues)

Business Metrics

  • Reduce avg ticket resolution time by 50%
  • 80% of common issues diagnosable without manual queries
  • 100% of customer integration configs documented
  • Support team adoption >90%
  • User satisfaction rating >4.0/5.0

Security & Compliance

Security Features

  • Read-Only Database Access: All connections are read-only to prevent data modification
  • Credential Encryption: All credentials stored in AWS Secrets Manager with encryption at rest
  • HTTPS Only: All communication encrypted in transit
  • Session Management: Automatic timeout after 60 minutes of inactivity
  • Audit Logging: All data access is logged for compliance
  • Role-Based Access: Separate permissions for staging vs. production access
  • 2FA Option: Two-factor authentication for production environment access
